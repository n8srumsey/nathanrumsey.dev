A learning project built to understand distributed consensus from first principles. The goal was to implement Raft closely enough to pass the MIT 6.824 lab tests without copying the reference implementation.

## Architecture

The store separates the consensus layer (Raft) from the state machine (the KV store itself). Raft handles log replication and leader election; the KV store applies committed log entries to an in-memory map. The two communicate through a single channel — the state machine never touches Raft internals.

```go
// LeaderElection initiates a new election round.
func (r *Raft) LeaderElection() {
    r.mu.Lock()
    r.currentTerm++
    r.votedFor = r.id
    r.state = Candidate
    term := r.currentTerm
    r.mu.Unlock()

    votes := 1
    var mu sync.Mutex

    for _, peer := range r.peers {
        go func(peer string) {
            ok, granted := r.sendRequestVote(peer, term)
            if !ok || !granted {
                return
            }
            mu.Lock()
            votes++
            if votes > len(r.peers)/2 {
                r.becomeLeader(term)
            }
            mu.Unlock()
        }(peer)
    }
}
```

## Persistence Layer

Log entries and voting state are persisted to disk using a write-ahead log before any response is sent. This ensures that a crashed node can recover its committed state after restart without needing to replay the full log from the leader.

```go
// persist writes durable state to the WAL before returning.
func (r *Raft) persist() {
    var buf bytes.Buffer
    enc := gob.NewEncoder(&buf)
    enc.Encode(r.currentTerm)
    enc.Encode(r.votedFor)
    enc.Encode(r.log)
    r.persister.SaveRaftState(buf.Bytes())
}
```

The persister interface is deliberately thin — in tests it's an in-memory implementation; in production it wraps a simple append-only file.
