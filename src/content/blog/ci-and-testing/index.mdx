A CI pipeline should be fast and specific. If everything runs in one job, a slow e2e suite blocks the fast unit tests from giving you feedback. This site runs four jobs in parallel, each catching a different class of problem.

![CI pipeline overview](https://picsum.photos/seed/ci-pipeline/800/400)

## The Four Jobs

**Typecheck** runs `astro check`, which runs the Astro TypeScript checker across all `.astro`, `.ts`, and `.tsx` files. This catches type errors that ESLint misses, including errors in frontmatter and component props.

**Lint** runs `eslint .` with a flat config targeting TypeScript rules on `.ts`/`.tsx` and the Astro plugin on `.astro`. These are kept separate because the two parsers don't compose cleanly.

**Unit** runs Vitest against co-located test files. Currently covers `slugs.ts` and `readingTime.ts` — pure utility functions with no framework dependencies.

**E2e** builds the site and runs Playwright against `npx serve dist/`. This is the integration check: does the built output actually render the right content at the right URLs?

```yaml
# .github/workflows/ci.yml (simplified)
jobs:
  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24' }
      - run: npm ci
      - run: npm run check

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24' }
      - run: npm ci
      - run: npm run lint

  unit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24' }
      - run: npm ci
      - run: npm test

  e2e:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24' }
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      - run: npm run test:e2e
```

## Why `serve` Instead of `astro preview`

The Playwright web server uses `npx serve dist/` rather than `astro preview`. The reason: `astro preview` does not serve plugin-generated static assets — the sitemap files produced by `@astrojs/sitemap` return 404 from `astro preview`, but serve correctly from a generic static file server. Since this is a fully static site with no SSR or middleware, `serve` faithfully represents what any static host will deliver.

```typescript
// playwright.config.ts
webServer: {
  command: 'npx serve dist/ --listen 4321',
  url: 'http://localhost:4321',
  reuseExistingServer: !process.env.CI,
},
```

## What Each Job Doesn't Catch

Typecheck won't catch runtime errors in content files — those fail at build time. Lint won't catch logical bugs. Unit tests don't cover component rendering. E2e tests don't prove accessibility — that's a separate `a11y.spec.ts` suite using axe-playwright, which runs in CI via `npm run test:a11y`.
