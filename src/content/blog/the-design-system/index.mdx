Every design decision on this site comes back to one constraint: it should feel like a document, not an application. That means high information density, restrained color use, and clear typographic hierarchy. Here's how the design system is built.

## Tokens

All design values live in a single `@theme {}` block in `src/styles/global.css`. Tailwind v4 reads these as its token source — no `tailwind.config.ts` needed.

```css
@theme {
  --color-background: #0d0d0f;
  --color-surface: #141416;
  --color-surface-raised: #1a1a1d;
  --color-nav: #111113;
  --color-border: #2a2a2e;
  --color-foreground: #e8e8ec;
  --color-muted-foreground: #888892;
  --color-primary: #7c8cf8;
  --color-primary-subtle: rgba(124, 140, 248, 0.08);

  --font-sans: 'Inter', system-ui, sans-serif;
  --font-mono: 'JetBrains Mono', 'Fira Code', monospace;

  --shadow-md: 0 2px 8px rgba(0, 0, 0, 0.4);
  --shadow-lg: 0 4px 24px rgba(0, 0, 0, 0.6);
}
```

These map directly to Tailwind utilities: `bg-background`, `text-muted-foreground`, `shadow-md`, and so on. No magic strings anywhere in component markup.

## Two-Surface Layout

The layout uses two distinct background layers. The full-width nav sits on `bg-nav`, and the content column floats above it on `bg-surface` inside a `max-w-3xl` container. Cards and list items use `bg-surface-raised` to lift off the surface without needing a drop shadow.

```astro
---
// BaseLayout.astro (simplified)
---
<body class="bg-background">
  <header class="bg-nav border-b border-border">
    <nav>...</nav>
  </header>
  <div class="max-w-3xl mx-auto my-8">
    <div class="bg-surface rounded-xl shadow-lg">
      <main class="px-8 py-10">
        <slot />
      </main>
    </div>
  </div>
</body>
```

The dot-grid pattern on the body background is a pure CSS trick — a `radial-gradient` repeating at `24px` intervals, using a semi-transparent foreground color as the dot color. No image assets involved.

## Typography

Body text uses Inter at 400/500/600 weight. Monospace text (dates, tags, metadata, code) uses JetBrains Mono. The distinction is consistent: anything data-like or code-adjacent is monospace, prose is sans-serif. This creates a clear visual hierarchy without relying on color alone.
